**80% Calm, 20% Motion**:
- Effects are subtle, not overwhelming
- Motion serves purpose, not decoration
- Breathing animations, not frantic movement
- Elegant transitions, not jarring changes
- Premium feel, not game-like chaos

## 1.3 Identity-First Architecture

**Player Identity**:
- Owned by PlayerId only
- Never TeamId
- ApplicationUserId for session isolation
- AssetId for asset identification

**Team Identity**:
- Visual assets owned by PlayerId inventory
- Visibility depends on current team ownership
- If player leaves team, assets disappear from team visual identity
- TeamId only for team association, not ownership

## 1.4 Living Asset Philosophy

**Static → Living Transformation**:
- Static PNG → Animated procedural rendering
- Static emblem → Living emblem with cinematic behaviors
- Static background → Dynamic ambient effects
- Static frame → Rank-based living effects

**Procedural Only**:
- All animation is procedural
- All particles are generated
- All timelines are code-defined
- No sprite sheets, no frame sequences
- No animated PNG, no GIF, no Lottie, no video

**Behavior Over Animation**:
- Every visual asset behaves like a living object
- Not merely "play an animation"
- Every asset has: Idle → Awareness → Action → Cooldown → Return to Idle
- Exactly like a living creature
